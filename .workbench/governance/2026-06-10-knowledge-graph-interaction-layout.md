# 2026-06-10 Knowledge Graph Interaction Layout

## 背景

用户要求优化知识图谱视觉与交互：

- 圆点和文字整体缩小。
- 浅色主题下连线需要更清楚。
- 圆点统一使用暖橙色 `#dc7d57` 系列，并按层级变化深浅。
- 支持鼠标滚轮缩放画布，支持按住画布或圆点拖动画布。
- 点击非文件夹圆点后关闭弹窗，并在文件编辑区打开对应文件。
- 圆点和文字之间不能重叠。

## 本轮变更

- 知识图谱 SVG 从漂移动画改为可缩放、可拖拽的视口层。
- 新增图谱节点点击打开文件链路，复用现有编辑器文件打开逻辑；文件夹节点不触发打开。
- 布局算法改为按知识库相对层级分环，并加入边关系牵引和碰撞松弛；节点圆点和文字估算宽度共同参与避让。
- 节点统一改为暖橙色系，按相对层级改变深浅；连线在浅色主题下加深。
- 更新项目地图中知识图谱交互口径。

## 验证

- `npm run build` 通过。
- Vite 本地页面 `http://127.0.0.1:5173/` 可加载，知识图谱入口可打开弹窗，画布区域具备抓手光标和禁用触摸滚动样式。
- 纯浏览器环境不是 Tauri 容器，`invoke` 不可用，因此未在浏览器内触发真实知识库数据和点击文件打开；该链路通过前端编译和复用现有 `wridian_open_file` 编辑器打开函数验证。

## 回退依据

- 图谱交互集中在 `src/App.tsx` 的 `KnowledgeGraphDrawer`、`buildKnowledgeGraphLayout` 和相关 helper。
- 视觉样式集中在 `src/App.css` 的 `.knowledge-graph-*` 与 `.graph-*` 规则。
- 如需回退，可恢复旧的静态 SVG 漂移动画层，并移除 `onOpenFile` 传递和 `openFilePath` 抽取。
