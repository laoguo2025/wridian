# 2026-06-09 Memory Tree Editor Panel

## 背景

用户要求记忆树弹窗内点击标签后直接编辑对应本地 Markdown 文件，编辑窗高度和树一致，并按标签所在侧决定编辑窗显示在左侧或右侧；点击其他位置时自动保存并收回编辑窗；右上角“记忆文件夹”按钮需要打开本地文件夹。

## 变更

- 记忆树标签选择改为携带编辑窗方向：右侧四个标签打开左侧编辑窗，其余主干和左/中标签打开右侧编辑窗。
- 编辑窗改为绝对定位在树画布内，顶部和底部与树区域对齐，textarea 填满剩余高度。
- 点击树画布空白处会保存当前草稿并收回编辑窗。
- 关闭弹窗前会保存当前草稿。
- 保存仍沿用 `wridian_save_memory_tree_file`，该命令只允许写记忆树或知识库内 Markdown 文件。

## 验证

- `npm run build` 通过。
- `cargo check --manifest-path src-tauri\Cargo.toml` 通过。
- 内置浏览器打开 `http://localhost:1420`，确认记忆树弹窗可打开，且只显示主标签，没有二级子标签。
- 浏览器预览缺少 Tauri invoke，因此无法在网页预览中读取真实记忆文件；真实文件写入路径通过后端命令约束和构建检查验证。

## 回退

回退 `src/App.tsx` 中的记忆树选择/保存逻辑和 `src/App.css` 中的 `.memory-node-detail` 定位样式即可。
