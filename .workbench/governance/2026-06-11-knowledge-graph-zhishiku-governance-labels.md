# 知识图谱 zhishiku 治理标签优化

## 原因

用户确认知识关系应该展示在图谱里，但要求图谱治理口径结合 `zhishiku-skill`，并且连线关系标签不能显示英文。

## 变更

- 将图谱治理视图文案改为 `zhishiku-skill` 的质量闸门和体检动作口径：缺编译来源、关联索引空、采纳未沉淀、孤岛待归档、重复待合并、高频老化。
- 在图谱顶部补充只读健康提示，明确质量闸门和可执行治理动作。
- 将固定 frontmatter 关系、正文链接和节点类型展示为中文。
- 未命中固定协议的动态 frontmatter 字段在连线标签上显示为中文“扩展关系”，避免把英文内部字段名直接展示给用户。
- “关联索引空”视图按知识卡出链为空判定，更贴近知识卡自身缺关联的治理含义。

## 验证

- 已运行 `npm run build` 验证前端构建通过。
- 已使用内置浏览器刷新 `http://127.0.0.1:4180/` 验收真实图谱组件：治理 tab、质量闸门、体检动作均显示中文；canvas 连线标签显示“关联”“编译来源”“已沉淀”等中文关系。

## 回退

回退 `src/knowledge/KnowledgeGraphDrawer.tsx`、`src/App.css` 以及本治理记录即可恢复本次 UI 文案和标签行为；关系解析和文件修复逻辑未变。
