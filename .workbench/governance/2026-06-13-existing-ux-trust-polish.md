# 2026-06-13 existing UX trust polish

## 背景

用户要求基于 Obsidian 帮助站启发，只优化 Wridian 现有功能，不新增功能。目标收口为三项：信任感、可理解、少打扰。

## 变更

- 对话消息的上下文弹层补充只读快照说明，强调上下文只用于生成回复，不会自动改正文。
- 正文 inline diff 的顶部汇总和单处按钮补充确认语义，明确确认后才写入正文，取消会保持原文。
- 知识图谱体检从“问题/一键修复”口径改为“需要关注、需要判断、处理低风险项”，避免把语义治理误导成自动改文件。
- 模型设置说明测试与保存的区别，并把常见测试失败转换为用户可操作的检查方向。
- 文件树回收站菜单补充系统回收站提示，避免误解为永久删除。

## 验证

- `npm run build` 通过。
- `npm run dev -- --host 127.0.0.1` 因固定 Tauri 端口 `127.0.0.1:1420` 返回 `EACCES`，未能启动开发服务。
- `npm run preview -- --host 127.0.0.1 --port 4173` 返回 HTTP 200，并用 Playwright CLI 截图确认主界面可渲染；普通浏览器预览因没有 Tauri IPC，会显示既有 `invoke` 读取错误，不代表桌面运行链路已验证。

## 回退依据

本轮只改前端文案、状态解释和少量辅助样式，不改变数据结构、文件操作协议、知识体检算法或模型请求链路。若需回退，可还原本记录对应提交。
