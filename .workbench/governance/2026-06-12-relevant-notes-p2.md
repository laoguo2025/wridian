# 2026-06-12 Relevant Notes P2

## 目标

接通已有后端 `wridian_find_relevant_notes`，在右侧对话区为当前稿件提供相关知识和作品记忆入口。

## 变更

- 前端新增 `RelevantNote` 类型，与后端 camelCase 返回结构对齐。
- 当前可编辑稿件内容变化后，前端延迟调用相关内容检索，限制首版展示 8 项。
- 对话区新增紧凑“相关内容”区域，按“知识 / 作品”分组展示。
- “加入”复用现有文件上下文 pill 读取链路，不写入创作记忆树，不改变知识图谱数据。
- “打开”复用现有文件打开链路。

## 边界

- 未改后端打分和扫描算法。
- 未把 Relevant Notes 切到 Metadata Index。
- 未扩展技能工作流或知识卡生产流程。

## 验证

- `npm run build` 通过。

## 回退

回退本次提交即可移除前端展示和调用；后端已有命令不受影响。
