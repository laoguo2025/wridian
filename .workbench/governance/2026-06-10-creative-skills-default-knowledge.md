# 技能管理入口与默认知识库

## 变更原因

- 顶部工具区需要在知识图谱右侧增加“技能管理”入口，承载知识库运维、作品拆解、知识卡提炼和大神蒸馏等技能管理。
- 知识库需要安装后有默认根目录和初始分类，但分类不能成为不可变协议；用户可以增、改、删，知识库运维 skill 后续按实际目录体检和修正。

## 变更范围

- 前端增加技能管理图标、弹窗和启用开关状态。
- 后端默认知识库根目录改为 `D:\Wridian知识库`，无 D 盘时回落到应用数据目录。
- 默认分类只在知识库根目录不存在时播种，避免用户改名、删除或清空分类后被重新创建。
- 知识图谱、对话上下文和记忆树知识卡入口读取解析后的当前知识库根目录。

## 验证

- `npm run build` 通过。
- `scripts/cargo-msvc.ps1 test --manifest-path src-tauri\Cargo.toml --lib` 通过，18 项测试全绿。

## 回退依据

- 如需回退默认知识库策略，还原 `src-tauri/src/runtime.rs` 的默认根目录与播种逻辑，并恢复 `workspace.rs`、`knowledge_graph.rs`、`cocreation.rs`、`memory.rs` 中只读取显式选择知识库的逻辑。
