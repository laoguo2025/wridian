# Medium Priority Audit Fixes

## 原因

修复审查后继续暴露的中优先级问题：记忆树内容编辑窗切换/关闭未等待异步保存，记忆树保存路径只校验父目录导致边界不够硬，相关稿件召回读取失败会被当作空内容静默吞掉。

## 变更

- 记忆树编辑窗保存改为返回成功/失败；切换节点、关闭编辑窗和关闭记忆树前会等待保存完成。
- 保存失败时保留当前编辑态并显示错误，不再先切走或关闭。
- 保存中禁用记忆树节点、叶子、关闭和候选叶动作，避免重复切换。
- 后端记忆树保存要求绝对路径，拒绝允许根外目标，拒绝符号链接 Markdown 文件，并校验已存在目标的规范路径仍在记忆树或知识库内。
- 相关稿件召回读取候选文件失败时返回明确错误，不再把失败文件当空文件参与排序。
- 增加 Rust 回归测试覆盖记忆树越界写入拒绝和相关稿件读取错误传播。

## 验证

- `npm run build`
- `powershell -ExecutionPolicy Bypass -File scripts\cargo-msvc.ps1 test --manifest-path src-tauri\Cargo.toml`

## 回退

如需回退，可恢复 `src/App.tsx` 的非等待保存逻辑以及后端旧路径/读取处理；但会重新引入切换丢草稿、保存边界偏弱和召回静默失败风险。
