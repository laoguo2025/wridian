# 2026-06-12 正文选区添加到对话交互修正

## 背景

用户指出“添加选区到输入框”不应常驻在编辑页顶部，也不应自动向对话输入框写入“请修改这段。”。期望交互是选中正文后，在选区上方浮出“添加到对话”按钮，点击后只把选区作为片段胶囊加入对话输入框。保存状态应移动到正文标题行右侧，减少顶部留白。

## 变更

- 删除编辑页顶部常驻“添加选区到输入框”按钮。
- 选中正文后根据浏览器选区矩形显示浮动按钮“添加到对话”。
- 点击浮动按钮只添加 selection pill，不自动改写输入框文本。
- “已保存”等保存状态移动到正文标题行右侧。

## 验证

- `npm run build`：通过。

## 回退

回退 `src/App.tsx`、`src/App.css` 本轮改动即可恢复旧顶部按钮行为。
