# Workbench Visual Polish

## 背景

用户要求继续优化首屏视觉：浅色主题不要太白、弱化分栏拖拽条、右侧对话默认更宽且输入区更轻、创作记忆树节点更精致、知识图谱重新定义分类/知识卡/关系边视觉语言。

## 变更

- 浅色主题 token 从纯白改为柔和纸灰和暖中性色，保留单一 Wridian 暖橙作为强调色。
- 分栏拖拽条默认改为极淡细线，仅 hover/focus 时出现暖橙反馈。
- 右侧对话默认宽度从 292px 提升到 332px，上限提升到 460px；输入框、未配置模型状态和发送按钮改为更轻的控件层级。
- 创作记忆树节点从白色标签改为半透明暖纸标签，并加入轻微高光和暖橙边界，降低贴纸感。
- 知识图谱新增图例，分类节点、知识卡节点、强关系边、引用边在绘制权重和样式上区分。

## 验证

- `npm run build` 通过。
- 使用本机 Chrome headless 抓取 `http://127.0.0.1:5173` 首屏截图，确认浅色主题从纯白降为柔和纸灰、分栏拖拽条默认弱化、右侧对话区默认更宽且输入区控件更轻。纯浏览器环境下右侧对话区仍会出现 Tauri `invoke` 报错，属于非桌面运行限制，不影响本次视觉验证。

## 回退

若浅色主题过暖或图谱图例干扰浏览，可先回退 token 与图谱图例样式，保留右栏宽度和拖拽条弱化这两个低风险优化。
