# 提炼全流程

目标：把案例拆解产物转成能被 AI 稳定调用、能在真实创作任务中复用的知识卡。知识卡不是资料摘要，每张卡都按一个小 skill 验收。

## 核心原则

- **只写 S 级**：S 级才能进入 `03-07`；A 级留在综合报告「待复审候选」；B 级丢弃。
- **先候选再评级**：不要边写边凑卡。先列候选、证据、落点、重复关系，再按闸门评级。
- **来源可回溯**：`编译来源` 必须精确到笔记、报告、场景或候选条目；不能只写作品名。
- **卡片必须 skill 化**：入库卡必须有输入、处理逻辑、输出、使用场景和失效边界。
- **宁缺毋滥**：单部作品通常 3-8 张 S 级，长篇大作最多 15 张。零新卡但更新旧卡是合法结果。
- **旧卡先审计**：同机制已有卡优先追加来源或更新关联，不重复建同义卡。
- **低频经验下沉**：主流程只保留质量闸门和常用路径；具体格式、更新模式和批量案例见 `references/`。

## Wridian P4 可执行工作流契约

- 输入：确认合格拆解报告、A 级候选、已有 `03-07` 知识卡和用户要新建还是更新。
- 扫描范围：只读取当前知识库内相关拆解、旧卡和调用记录；旧卡审计先搜机制关键词和来源。
- 产物：S 级知识卡写入 `03-07`，A/B 候选处理结果回写案例 README 或报告，调用表现写入 `00知识库治理/调用记录台账.md`。
- 写入：新增或更新卡片、README 和台账必须通过 Wridian `fileOperations` 返回，不能只口头描述。
- 质检：每张入库卡必须具备输入、处理逻辑、输出、使用场景、失效边界、可定位编译来源和非空关联索引。
- 回滚：错误卡片通过相对路径追加修正、恢复旧版本或移到系统回收站；不得永久删除正式知识卡。

## Phase 0：知识库定位

知识库根目录优先来自 Wridian 当前库；脚本化执行时由参数或 `WRIDIAN_KNOWLEDGE_ROOT` 传入。无法定位时先询问用户路径。所有产物路径以 `$ROOT` 表示。

## Phase 0.5：上游拆解入口闸门

如果输入来自 `chaijie-skill`，必须先检查拆解产物质量。以下任一缺失，停止入库，只能补拆解或保留待复审候选：

- 有覆盖声明：阅读方式、覆盖范围、缺失范围、结论权限。
- 关键判断有证据锚点。
- 知识卡候选已分 A/B/C；只有 `A 可提炼候选` 能进入本 skill。
- 抽样、缺章、低质源只产出局部观察或待复核候选。

## Phase 1：输入类型判断

```text
输入是？
├─ 已完整拆解案例 → 直接进入 Phase 4
├─ 只有部分笔记/报告 → 先补齐证据与候选等级
├─ 原始小说/剧本/视频/图片 → 转 chaijie-skill
├─ 多部同作者拆解 → 合并候选，做交叉验证
└─ 横向对比材料 → 直接做候选矩阵，再评级
```

快速路径条件：案例目录含 README、分析笔记、综合报告、审美反应或蒸馏候选。缺一项时可以提炼，但必须在报告里标注样本边界。

## Phase 2：审美校准

新案例提炼前，读取 `06写作技法/` 中现有意识卡的一句话法则，用来校准高光、低光和边界。若目录为空，跳过。

## Phase 3：综合候选池

从分析笔记、综合报告、审美反应、蒸馏候选中汇总候选。每个候选必须写清：

```markdown
### 候选名
- 来源锚点：
- 核心机制：
- 建议落点：03/04/05/06/07
- 可迁移用法：
- 可能重复的旧卡：
- 初评：S/A/B
- 降级原因：
```

候选不足时，不为了凑数入库；候选过多时，先合并同义机制。

## Phase 4：知识卡 skill 化闸门

每张 S 级卡必须回答五个问题，否则最高只能 A：

| 问题 | 最低要求 |
|---|---|
| 输入是什么 | 说明什么任务、场景、素材或创作阶段会调用它 |
| 处理逻辑是什么 | 有步骤、公式、判断顺序、检查清单或组合路线 |
| 输出结果是什么 | 说明能产出诊断、桥段、人物方案、结构方案、素材还是审美判断 |
| 谁买单/用于什么场景 | 写清使用者、项目阶段、媒介或业务场景 |
| 失效边界是什么 | 说明不适用题材、媒介、篇幅、阶段或误用风险 |

以下内容不得作为 S 级入库：

- 剧情复述、人物小传、世界观介绍。
- 只解释原作品，不能迁移到新创作。
- 空泛审美判断、鸡汤式写作建议。
- 没有证据锚点，或证据只支持局部观察。
- 多个技法混成一张卡，调用时无法执行。

## Phase 5：S/A/B 评级

| 等级 | 处理 | 判定 |
|---|---|---|
| S | 写入或更新 `03-07` | 可复用、可调用、证据强、边界清楚、命名准确；换作品仍有指导价值 |
| A | 留在报告「待复审候选」 | 有启发但证据不足、步骤不清、场景窄、仍像案例摘要 |
| B | 丢弃 | 剧情总结、概念空泛、重复旧卡、无法调用、证据不可定位 |

硬门槛：来源可定位、可调用、可迁移、非重复、有边界。任一条不满足，最高 A。

常见降级信号：

- 人设特征而非技法：不能写出操作步骤。
- 过于通用：十部不同作品都能套，缺少独特生成力。
- 单作孤证：只有一段笔记支持，不能升为稳定机制。
- 题材依赖过强：换媒介或题材即失效，且未写边界。

## Phase 6：03-07 落点标准

| 板块 | S 级最低标准 |
|---|---|
| `03故事模型` | 长期运行引擎；说明驱动力、阶段推进、核心矛盾、适用题材、失效条件 |
| `04人物原型` | 可复用人物位置/关系功能/精神内核；能指导新角色设计 |
| `05情节方程` | 桥段公式清晰：前提→动作→反应→情绪结果；frontmatter 必须有 `情绪` |
| `06写作技法` | 可执行技法、组合流程或审美法则；步骤/顺序清楚，有使用场景和判断边界 |
| `07综合素材` | 高密度可调用素材；短、准、有使用方式，不是大段资料搬运 |

具体 frontmatter 和正文结构读取 `references/card-formats.md`。

## Phase 7：写入或更新

执行顺序：

1. 搜索 `03-07` 旧卡：用作者名、作品名、机制关键词分别查，不只搜文件名。
2. 判断关系：同机制追加来源；角度不同且调用逻辑不同，才新建。
3. 逐张写入 S 级卡；每写 3-5 张立刻检查 frontmatter。
4. A 级候选写入综合报告「待复审候选」；B 级写明丢弃原因。
5. 更新本案例 README：写明新建、更新、待复审、丢弃数量。
6. 提醒或更新 `00知识库治理/调用记录台账.md`，记录本轮调用表现和后续进化方向。

frontmatter 铁律：

```yaml
---
板块: {03故事模型/04人物原型/05情节方程/06写作技法/07综合素材}
类型: {对应类型}
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: {精确来源}
关联索引: [至少一条]
调用阶段: {对应阶段}
title: {标题}
tags: [{标签}]
调用摘要: {80-150字说明怎么用}
updated: YYYY-MM-DD
---
```

补充字段：

- `05情节方程` 必须有 `情绪`。
- `06写作技法` 必须有 `一句话法则`、`案例数`、`颗粒度`。
- `07综合素材` 必须有素材类型、调用方式或适用场景。

## Phase 8：交叉验证策略

### 首次作者

只有一部作品时仍可出 S 级，但必须更保守：

- 同一机制在 3 段以上笔记反复出现，才可视为作品内交叉验证。
- `tags` 加 `需跨作品验证`。
- 数量通常 3-5 张；中短篇通常 3-4 张。
- 只出现一次的好想法评 A。

### 同作者多书

- 多书互证的候选优先升 S。
- 高度重叠候选合并成一张卡。
- 第 2 部作品可追加来源并移除 `需跨作品验证`。
- 第 3 部及以后主要追加来源、升级 A 级候选、识别作者级缺陷。
- 纯验证零新卡是合法结果。

### 跨作者验证

新作者验证旧卡时，优先给旧卡追加 `编译来源`，不移除 `需跨作品验证` 标签。只有机制独特、公式不同、调用输出不同，才新建卡。

更多更新模式见 `references/card-update-patterns.md`。

## Phase 9：审美意识提炼

每次提炼记录 3-5 个审美反应：

```text
[高光] {具体场景} -> 审美理由：{好在哪}
[低光] {具体场景} -> 审美理由：{差在哪}
```

规则：

- 意识是品味法则，不是操作技法。
- 强化已有意识：追加来源、案例数、正文案例。
- 找不到对应：先作为候选；累计 3 个案例后再创建 `06写作技法` 卡。

## Phase 10：收工自检

完成前检查：

- 新增/更新卡 frontmatter 可解析，`评级: S`，`评审状态: S级保留`。
- `编译来源` 文件存在，`关联索引` 非空且尽量可定位。
- 内容忠实度抽查：抽查至少 3 张新卡回源验证关键断言。
- 跑全库 health-check；历史旧账与本轮问题分开汇报。
- 统计本轮 S 新建、S 更新、A 待复审、B 丢弃数量。

命令：

```bash
bash "$ROOT/health-check.sh"
```

若只剩旧卡 YAML 数组来源误报，手动验证来源存在后注明，不为了过检破坏多来源证据链。
