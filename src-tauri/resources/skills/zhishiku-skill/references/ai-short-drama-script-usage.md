# AI短剧剧本创作时如何调用Wridian知识库

## 触发场景

当用户要求写、重写、改写、续写、修订 AI短剧 / 竖屏短剧 / 短剧剧本，尤其用户提到“知识库”“利用起来”“剧本不是小说”“每集多少字”“卡点”“AI短剧格式”时，必须先调用Wridian知识库，而不是凭空写散文化剧情。

## 输出格式硬要求

AI短剧剧本不是小说，不要写成散文段落。默认使用：

```text
第X集
X-1：【场景名·时间·内/外】
人物：角色A、角色B
△首帧冲击/镜头/动作/音效（直接写清画面，不另设AI生成提示模块）
角色名（语气/情绪）：短台词
△动作或镜头推进
```

必须包含：

1. 集号、场号连续，不缺集跳场。
2. 每场有人物栏。
3. 动作描述使用白三角 `△`。
4. 台词短、直给，避免长篇内心独白。
5. 每集开头有首帧冲击或强冲突。
6. 每集结尾有卡点：情绪最高点 + 新威胁 / 新问题。
7. 画面描写要服务生成一致性：角色锚定、服装、光影、道具、音效、特效都应写进正文动作/镜头里，不另设“AI生成提示”模块。
8. 关键视觉锚点反复出现，并明确停留时长或特写方式。
9. 如果用户要求“每集1000字，含标点”，需统计每集正文去空白后的字符数，并在文档内给表格。

## 写作工作流

1. 先扫描项目文件，确定题材、人物、避险要求、必保卡点。
2. 检索知识库短剧方法论和同类短剧案例，不要跳过。
3. 提炼本次可调用的规则，如：
   - 黄金三章 / 前3集讲透优势。
   - 三集一个小高潮，十集一个大高潮。
   - 前3秒强钩子。
   - 前十集不水剧情。
   - 每场戏必须有情绪价值或推进主线。
   - 卡点=情绪最高点+新威胁。
   - 爽感与代价同步。
4. 建角色卡和视觉锚点，再写正文。
5. 用脚本或工具统计每集字数，写入表格。
6. 抽读开头、第10集/付费卡点，确认不是小说化。

## 常见误区

- 只写“剧情梗概”或“小说段落”，没有场号、人物、△动作。
- 额外设置“AI生成提示”模块，把画面说明和正戏拆开；应直接在正文动作与镜头里写清画面。
- 写了漂亮故事，但 AI 无法生成：人物外观不固定、场景光影不清、动作太抽象。
- 职业流短剧把专业装备写成万能外挂；涉及消防救援时，先查看 `references/fire-rescue-technical-notes.md`，专业爽点应来自“工具限制下的判断与训练”，不是工具替代判断。
- 长对白解释设定，缺少画面动作。
- 开头先铺世界观，没在前3秒给冲突。
- 卡点只是“结束”，不是“情绪最高点+新威胁”。
- 知识库已有短剧方法论却没有调用，导致输出偏网文/小说。
