# 源文件归一化策略

拆解前先把复杂格式转为可稳定读取、可切分、可追溯的 `md/txt` 中间文件。原文件必须保留。

## 支持格式

| 原格式 | 中间格式 | 推荐工具 | 说明 |
|---|---|---|---|
| `.txt` | `.txt` | 无 | 原样使用 |
| `.md` | `.md` | 无 | 原样使用 |
| `.docx` | `.md` | pandoc | 保留标题、列表、脚注 |
| `.epub` | `.md` 或 `.txt` | pandoc | 方法论书籍转 md，小说可转 txt |
| `.mobi` | `.epub` → `.md/.txt` | calibre `ebook-convert` + pandoc | 不直接依赖 pandoc 读 mobi |
| `.pdf` | 按页 `.txt/.md` | pdftotext / OCR | 优先保留页码锚点 |
| 扫描版 `.pdf` | OCR 后 `.txt` | ocrmypdf 或其他 OCR | 转出空文本时不得继续拆解 |
| 图片 | OCR 后 `.txt` | OCR | 只保留可引用文字 |
| 音视频 | transcript `.txt` | 转录工具 | 后续按文本处理 |

## 工具原则

`zhishiku-skill` 不内置 pandoc、calibre、OCR 本体，只声明调用协议：

1. 检测本机是否存在所需工具。
2. 存在则调用。
3. 不存在则提示缺依赖，并给出可替代路径。

Pandoc 适合 `docx/epub/html/md`，不作为 PDF/OCR/mobi 的万能入口。

## 归一化目录

推荐在案例目录内保存原件、中间文件和记录：

```text
02拆解报告/{媒介类型}/{作品名}/_source/
├── 原文件.docx
├── 原文件.md
├── 分卷/
│   ├── part-001.md
│   ├── part-002.md
│   └── manifest.md
└── 转换记录.md
```

## 转换记录模板

```markdown
# 转换记录

| 字段 | 值 |
|---|---|
| 原文件 | xxx.docx |
| 原格式 | docx |
| 转换工具 | pandoc |
| 输出文件 | xxx.md |
| 分卷目录 | 分卷/ |
| 转换时间 | YYYY-MM-DD |
| 是否需要人工复核 | 否 |

## 抽查结果

- 章节是否完整：是/否
- 中文是否乱码：是/否
- 表格是否丢失：是/否/不适用
- PDF 页码是否保留：是/否/不适用
- 扫描件/OCR 风险：无/有
```

## 超长文件切分

转换后的 `md/txt` 如果字数或行数过大，不能一次性读完，必须先切分。

推荐阈值：

| 类型 | 切分阈值 | 推荐切分方式 |
|---|---:|---|
| 长篇小说 | 2-5 万字/份 | 优先按卷/章/事件段 |
| 方法论书籍 | 1-3 章/份 | 按章节结构 |
| 访谈/播客转录 | 1-2 万字/份 | 按话题或时间戳 |
| PDF | 20-50 页/份 | 按页码范围 |

切分原则：

- 不在高潮、论证链、表格中间切。
- 优先自然边界：卷/章/节/场景/页码。
- 每个分卷文件必须能独立读取。
- `manifest.md` 记录分卷顺序、来源范围、字数/行数。
- 后续拆解 worker 只读分卷文件，不直接吞整本大文件。

## 质量门槛

转换不是理解。进入拆解前必须抽查：

1. 开头、中段、结尾是否都有内容。
2. 标题/章节是否基本保留。
3. 中文是否乱码。
4. PDF 是否转出空文本。
5. 分卷是否连续、无明显缺页或缺章。

如果抽查失败，停止拆解，先修复转换或更换源文件。
