# 综合报告结构

综合报告属于 `02拆解报告` 的案例产物。它的作用不是替代知识卡，而是给后续 `03-07` 提炼提供证据和候选清单。

## 七章节模板

适用：长篇小说、中篇小说、剧本、影视、游戏剧情等叙事型案例。

| 章节 | 内容 |
|---|---|
| 一、全局结构分析 | 时间线表、叙事节奏分段、各卷/幕/章节分工，标注不可逆转折点 |
| 二、人物系统分析 | 核心角色的身份定义、完整弧光、叙事功能、关系结构 |
| 三、技法全景分析 | 汇总所有笔记技法，去重后按结构、人物、关系、博弈、世界构建等类别归纳 |
| 四、主题与世界观 | 2-5 条底层命题，每条有情节支撑；世界观、组织、规则、符号体系 |
| 五、跨案例对标 | 与其他已拆案例的技法、人物、命题对标，判断共性、差异和可融合卡 |
| 六、待提炼清单 | 按当前 `03-07` 目录列出所有知识卡候选 |
| 七、总结与建议 | 核心发现、可迁移创作建议、后续研究方向 |

## 待提炼清单格式

第六章必须按当前目录判断：

```markdown
## 六、待提炼清单

### 03故事模型
- {候选名}：来源 {笔记路径/报告章节}；理由；建议落点。

### 04人物原型
- {候选名}：来源；人物位置；可复用点；建议落点。

### 05情节方程
- {候选名}：来源；主情绪；触发结构；建议主分类。

### 06写作技法
- {候选名}：来源；可执行步骤、组合路线或审美判断；适用场景；是否已有3案例支撑。

### 07综合素材
- {候选名}：来源；素材类型；调用方向。
```

没有候选的目录写「本案例暂未发现稳定候选」，不要硬凑。

## 非叙事类报告变化

| 源类型 | 报告类型 |
|---|---|
| 方法论书籍 | 知识对接型：全书体系 → 方法论清单 → 与 `06/07/08` 的对接点 → 可补写卡 → 案例资产 |
| 公众号文章/营销文案 | 传播策略型：选题规律 → 钩子公式 → 情绪模式 → 转化链路 → 可提炼方程 |
| 图片系列 | 视觉叙事型：视觉母题 → 色彩弧线 → 构图与权力 → 可提炼视觉方程/审美意识/素材 |
| 短视频系列 | 流量机制型：账号定位 → 0-3 秒钩子公式 → 情绪模板 → 视听风格 → 短视频方程 |
| 横向对比 | 对比矩阵：维度 × 作品，通常直接写卡，无独立报告 |
| 混合素材包 | 归类报告：主题分组 → 关键素材 → 可提炼方向 |

## 报告质量标准

- 技法全景不复述剧情，必须写清「作者/导演做了什么」和「为什么有效」。
- 每个主题判断必须有情节、场景或原文支撑。
- 跨案例对标必须指出相同点和差异点。
- 待提炼清单必须能直接交给知识卡 worker 使用。
- 不确定的候选标注「待验证」，不要写成稳定结论。
