# 知识卡格式与评级标准

本文件定义 `tilian-skill` 写入 `03-07` 的格式与 S/A/B 评级。最终只写入或保留 S 级卡；A 级作为待复审候选留在报告；B 级丢弃。

## 总原则：只保留 S 级

| 等级 | 处理 | 判定 |
|---|---|---|
| S | 写入或保留在 `03-07` | 可复用、可调用、证据强、边界清楚、命名准确；换一个作品/项目仍有指导价值 |
| A | 不入库，留在报告「待复审候选」 | 有启发但证据不足、适用场景窄、表达还像案例摘要；需更多案例验证 |
| B | 丢弃 | 只是剧情总结、概念空泛、重复旧卡、无法调用、证据不可定位、没有迁移价值 |

硬门槛：任一条不满足，最高只能 A。

- 来源可定位：能回到具体笔记/报告/场景。
- 可调用：有明确使用场景，不只是解释作品。
- 可迁移：能从原案例抽出规则，迁移到新创作。
- 非重复：不能只是已有卡换标题。
- 有边界：知道什么时候不适用。

S 级数量控制：单部作品一次提炼通常只保留 3-8 张 S 级卡；长篇大作最多 15 张。宁缺毋滥。

## 全部卡共性要求

每张卡必须有标准 YAML frontmatter：

```yaml
---
板块: {目录名}
类型: {卡片类型}
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [至少一条]
调用阶段: {对应调用阶段}
title: {标题}
tags: [{标签}]
调用摘要: {80-150字说明怎么用}
updated: YYYY-MM-DD
---
```

铁律：

- `板块` 必须与所在目录一致。
- `关联索引` 不得为空。
- `编译来源` 必须精确到具体笔记、报告或条目。
- 禁止用 `|` 在一行内拼多个 YAML 字段。
- 案例提取卡必须能回连到 `02拆解报告` 案例产物。
- 只允许 S 级卡写入 `03-07`；A/B 候选不得入库。
- `评级` 必须为 `S`。
- `评审状态` 必须为 `S级保留`。
- 每次提炼先列候选，再评级，再写入。

## 03故事模型卡

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 解释整部作品或一类题材的长期运行引擎；能说明驱动力、阶段推进、核心矛盾、适用题材、失效条件 |
| A | 只解释单部作品，跨作品迁移还不稳；可作为模型候选 |
| B | 只是剧情梗概、题材标签、世界观介绍 |

适合：全篇故事运行模式、题材底层结构、长线叙事机制。

存储：

```text
03故事模型/{子分类}/《作品名》-{模型名}.md
03故事模型/{子分类}/通用-{模型名}.md
```

frontmatter：

```yaml
---
板块: 03故事模型
类型: 故事模型
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [相关卡片]
调用阶段: 故事构思
title: {模型名}
tags: [故事模型, {子维度}, {题材标签}]
调用摘要: {80-150字说明这个模型适合什么故事}
updated: YYYY-MM-DD
---
```

正文参考结构：

```markdown
## 一句话定义
## 核心运行逻辑
## 关键情绪节点
## 适用题材
## 与相似模型的区别
## 经典案例
```

## 04人物原型卡

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 提炼可复用的人物位置、关系功能、精神内核；能指导新角色设计 |
| A | 人物很鲜明但还停留在角色分析；迁移规则不够清楚 |
| B | 只是人物小传、性格词堆砌、外貌履历 |

适合：可复用人物位置、人物功能、角色关系结构。

当前只使用六类目录：

```text
04人物原型/男主
04人物原型/女主
04人物原型/男反
04人物原型/女反
04人物原型/女配
04人物原型/配角
```

不使用旧分类 `根`。

命名：

```text
04人物原型/{位置}/《作品名》-{位置}-{系名}.md
04人物原型/{位置}/通用-{编号}-{位置}-{系名}.md
```

frontmatter：

```yaml
---
板块: 04人物原型
类型: 人物原型
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [相关卡片]
调用阶段: 人物设计
title: {位置}-{系名}
tags: [人物原型, {位置}, {功能标签}]
调用摘要: {80-150字说明这个人物原型怎么用}
updated: YYYY-MM-DD
---
```

正文结构：

```markdown
## 皮（视觉锤）
## 肉（性格血肉）
## 骨（结构功能）
## 血（关系温度）
## 魂（精神内核）
```

系名用「系」，不用「型」。

## 05情节方程卡

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 一场戏或桥段内有清晰情绪触发公式：前提→动作→反应→情绪结果；必须标 `情绪` |
| A | 桥段有效但公式不够清楚；情绪结果可感但结构步骤松散 |
| B | 只是情节复述、没有观众情绪、没有结构步骤 |

适合：一场戏或一个桥段内完成的情绪触发公式。

当前只使用八类目录：

```text
05情节方程/01-心疼同情催泪
05情节方程/02-爽感热血
05情节方程/03-悬念反转
05情节方程/04-愤怒痛苦绝望
05情节方程/05-紧张博弈战术
05情节方程/06-关系情感
05情节方程/07-叙事结构节奏
05情节方程/08-情绪共鸣
```

分类看主要观众反应，不看表面题材。

frontmatter：

```yaml
---
板块: 05情节方程
类型: 情节方程
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [相关卡片]
调用阶段: 分场写作
title: {方程名}
tags: [情节方程, {主分类}, {情绪词}, {功能词}]
情绪: {主情绪}
调用摘要: {80-150字说明这个方程怎么用}
updated: YYYY-MM-DD
---
```

正文固定五块：

```markdown
## 情节功能
## 生效条件
## 基本结构
## 创作要点
## 经典案例
```

## 06写作技法卡：单点技法

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 单一可执行技法；步骤明确，能被作者马上拿去写某个场景、句段或桥段 |
| A | 技法方向对，但步骤还粗；需要更多案例校准 |
| B | 抽象建议、鸡汤、无法执行、多个技法混在一起 |

适合：可执行写作方法、流程、清单、训练法、拆解法、审校法。

存储：

```text
06写作技法/{子目录}/《作品名》-{技法名}.md
06写作技法/{子目录}/通用-{描述性名称}.md
```

frontmatter：

```yaml
---
板块: 06写作技法
类型: 单点技法
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [相关卡片]

title: {技法名}
tags: [单点技法, {分类标签}]
调用摘要: {80-150字说明什么时候用}

```

## 06写作技法卡：组合技法

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 多个技法或模块的组合路线；能说明调用顺序、协同关系、适用项目类型 |
| A | 有系统感但调用顺序不清；适用边界还不稳 |
| B | 单点技法误放、概念图口号、分类目录而非路线 |

适合：题材定位、市场判断、调用路由、创作系统、世界观系统。

存储按当前目录选择：

```text
06写作技法/{子目录}/《作品名》-{描述性名称}.md
06写作技法/{子目录}/通用-{描述性名称}.md
```

常用子目录包括但不限于：

```text
01-故事结构
02-人物塑造
03-系统书目
04-创作原理
09-市场定位
```

frontmatter：

```yaml
---
板块: 06写作技法
类型: 组合技法
状态: 已萃取
评审状态: S级保留
评级: S
编译来源: 02拆解报告/{类型}/{作品目录}/分析笔记-{卷/段}.md
关联索引: [相关卡片]
调用阶段: 创作规划
title: {图谱名}
tags: [组合技法, {分类标签}]
调用摘要: {80-150字说明这个图谱怎么调用}
updated: YYYY-MM-DD
---
```

正文按功能选择，不强行统一。路由卡优先写「调用场景、调用顺序、组合方式、避坑」。

## 06写作技法卡：审美法则

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 至少 3 个案例交叉验证的品味法则；说明高光/低光和审美判断边界 |
| A | 只有 1-2 个案例，先做候选意识，不入库 |
| B | 单次喜好、读后感、价值判断无场景证据 |

适合：审美判断、创作观念、底层品味法则。

存储：

```text
06写作技法/意识{编号}-{名称}.md
```

要求：

- 是审美法则，不是操作步骤。
- 至少 3 个案例交叉验证后再建卡。
- 验证案例必须具体到场景。
- 格式以 `06写作技法/` 现有卡片为准，并必须通过 health-check。

## 07综合素材卡

提炼标准：

| 等级 | 标准 |
|---|---|
| S | 高密度可调用素材：设定、道具、机构、术语、场景、金句；短、准、能直接填充创作 |
| A | 素材有趣但用途不明；需要补调用方式 |
| B | 大段摘录、资料搬运、没有调用方式 |

适合：道具、行话、场景、机构、传说、金手指、台词金句、世界观碎片。

## 分类决策

| 这个东西 | 归入 |
|---|---|
| 定义一种故事长期运行模式 | 03故事模型 |
| 定义一种人物模式或角色功能 | 04人物原型 |
| 一场戏内触发明确情绪 | 05情节方程 |
| 可执行方法、流程、清单 | 06写作技法 |
| 题材定位、系统图谱、调用路线 | 06写作技法 |
| 审美判断、品味法则、创作观念 | 06写作技法 |
| 设定、道具、行话、场景、金句 | 07综合素材 |

## 易混区分

| 易混对 | 区分 |
|---|---|
| 03 vs 06 | 模型回答「这类故事怎么跑」；技法回答「怎么写出这个效果」 |
| 05 vs 06 | 方程是一场戏的情绪触发；技法是跨流程操作方法 |
| 05 vs 09 | 方程是动作结构；素材是可调用元素 |
| 07 vs 08 | 图谱偏系统路线；审美意识偏判断和品味 |
| 04 vs 09 | 原型是人物功能；素材是人物细节 |
