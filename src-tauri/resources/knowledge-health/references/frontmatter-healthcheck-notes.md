# 03-07 frontmatter 体检与修复记录

本次会话验证的稳定经验：

## 1. 先判定是否真缺字段

03-07 旧卡大量“缺字段”并非字段缺失，而是 frontmatter 结束符没有独立成行，常见形态：

- `---## 一句话定义`
- `created: 2026-04-15---`
- `updated: 2026-05-07---`

修复原则：只把 `---` 拆成独立行，保留原字段值和正文，不重写分析判断。

## 2. 批量修复前必须备份

推荐备份目录：

```text
09文件归档/frontmatter修复备份-YYYY-MM-DD/
```

先备份，再批量改卡；不要直接覆盖旧卡。

## 3. health-check 的旧卡兼容点

旧卡 frontmatter 修复后，体检仍可能因为语义规则报错：

- `评审状态: S级保留`
- `评审状态: A级待复审`
- `编译来源` 为数组、带引号、带 `:行号` / `:起止行`

更稳妥的做法是让校验器兼容旧卡业务语义，而不是为了过检硬改卡片含义。

## 4. 抽查要看三件事

修复后至少抽查 3-5 张：

- frontmatter 起止 `---` 是否独立成行。
- 正文标题是否被吞掉。
- `created/updated` 值是否仍完整。

## 5. 批量补缺失字段的目录映射规则

当 health-check.sh 报"缺调用阶段"或"缺类型"时，可按目录自动补默认值：

| 目录 | 调用阶段 | 类型 |
|------|----------|------|
| 03故事模型 | 选题立项 | 故事模型 |
| 04人物原型 | 人物设计 | 人物原型 |
| 05情节方程 | 分场写作 | 情节方程 |
| 06写作技法 | 方法执行 | 写作技法 |
| 07综合素材 | 细节填充 | 综合素材 |

执行方式：用 Python 读文件 → 定位第二个 `---` → 在其前插入缺失字段行 → 写回。不要用 sed（中文路径和多行 frontmatter 容易出错）。

板块不一致修复：当 frontmatter 中 `板块:` 值与文件实际所在一级目录名不同时，直接替换为正确目录名。常见案例：旧版单点技法、组合技法、审美意识类板块应统一改为 `06写作技法`。

## 6. 旧目录残留归档

当发现 SCHEMA.md 规定之外的旧目录仍存在且含文件时：

- 归档路径：`09文件归档/旧目录归档-YYYY-MM-DD/{旧目录名}-原目录/`
- 直接 `mv` 整个旧目录，不逐文件操作。
- 编号版子目录（如 `04人物原型/01-男主`）的文件应迁入无编号版对应目录，迁完后 `rmdir` 空目录。

## 7. 编译来源路径修正

当 health-check 报"来源未定位"时，常见根因是 02拆解报告目录经过重组（加入了媒介类型/作者层级），但 03-07 卡片的 `编译来源` 字段仍指向旧路径。

典型案例：
- 旧路径：`02拆解报告/无限血核/分析笔记-无限血核-卷21.md`
- 实际路径：`02拆解报告/小说/蛊真人/《无限血核》/分析笔记-无限血核-卷21.md`

修复方式：
1. 先 `find` 确认实际路径结构。
2. 用 `sed -i ''` 批量替换旧前缀为新前缀（macOS 需 `''` 空备份参数）。
3. 若旧路径含合卷引用（如 `卷1~9`、`卷7~8`），而实际文件是单卷命名，用正则 `卷([0-9]+)~[0-9]+` → `卷\1` 映射到首卷。
4. 替换后重跑 health-check 验证归零。

注意：`关联索引` 字段也可能存在同类旧路径问题，修复时应一并检查。

## 8. 07综合素材补调用摘要与颗粒度

当 health-check 报"07缺调用摘要"或"07缺颗粒度"时：

- **调用摘要**：80-150 字，格式为"核心内容——适用于XXX创作场景"。需阅读卡片正文后撰写，不可机械复制标题。
- **颗粒度**：按正文字数判断——标准（≤200字）/ 偏长（200-500字）/ 待压缩（>500字）。

执行方式：Python 读文件 → 在 frontmatter 闭合 `---` 前插入两个字段 → 写回。07综合素材的卡片通常很短（12-15行），大多为"标准"。

## 9. 适用结论

这类修复属于低风险结构修复，适合纳入知识库总控的标准流程，不应与内容提炼混为一谈。批量补字段和旧目录归档均可在用户授权后自动执行，无需逐文件确认。
