# 目录结构

Wridian 知识库体检资源定义的默认知识库包含 10 个一级目录。

```text
Wridian知识库/
├── 00知识库治理/
├── 01原始资料/
├── 02拆解报告/
├── 03故事模型/
├── 04人物原型/
├── 05情节方程/
├── 06写作技法/
├── 07综合素材/
├── 08大神蒸馏/
└── 09文件归档/
```

## 00知识库治理

初始化时只放调用台账：`调用记录台账.md`。治理说明、目录职责、知识卡输入/逻辑/输出/边界、旧目录归档建议统一写在根目录 `知识库使用说明.md`。

## 01原始资料

存放待处理素材：小说 txt/epub、剧本、影视字幕、方法论书籍、文章、图片、短视频转录等。

规则：拆解完成后，源文件可移动到对应 `02拆解报告` 案例目录归档。

## 02拆解报告

存放作品拆解产物：分析笔记、综合报告、分卷文件、源文件归档。

推荐结构：

```text
02拆解报告/小说/{作者名}/《作品名》/
02拆解报告/电影/《作品名》/
02拆解报告/短剧/《作品名》/
02拆解报告/文章/《文章名》/
02拆解报告/方法论书籍/《书名》/
```

复杂格式和超长文件先进入 `_source/`：

```text
02拆解报告/{媒介类型}/{作品名}/_source/
├── 原文件
├── 转换后的.md或.txt
├── 转换记录.md
└── 分卷/
    ├── part-001.md
    ├── part-002.md
    └── manifest.md
```

拆解 worker 读取分卷文件，不一次性读取整本超长文件。

## 03故事模型

存放解释整部作品或一类题材长期运行引擎的 S 级知识卡。

## 04人物原型

存放可复用人物位置、关系功能、精神内核的 S 级知识卡。

子目录（SCHEMA 规定）：`男主/` `女主/` `男反/` `女反/` `女配/` `配角/`（无编号）。

## 05情节方程

存放一场戏或桥段的情绪触发公式。必须有 `情绪` 字段。

主分类（SCHEMA 规定，仅保留这 8 个）：`01-心疼同情催泪` / `02-爽感热血` / `03-悬念反转` / `04-愤怒痛苦绝望` / `05-紧张博弈战术` / `06-关系情感` / `07-叙事结构节奏` / `08-情绪共鸣`。

## 06写作技法

存放单一可执行写作技法、组合流程、调用图谱、创作系统和经过至少 3 个独立作品/项目验证的审美法则。

S 级要求：步骤明确、能马上用于场景/句段/桥段；组合类要写清调用顺序和协同关系；审美法则类要有具体高光案例、失效边界，并可指导未来创作判断。

## 07综合素材

存放高密度可调用素材：设定、道具、机构、术语、场景、金句。

## 08大神蒸馏

存放 `zhengliu-skill` 产出的作者小 skill。

当前实际结构（2026-05-15 扫描）：每位作者一个 `{拼音}.skill` 目录，共 16 位作者：changshuxin / chendong / fenghuo / huweidebi / jiayu / liulangdehama / maboyong / maibao / maoni / nanpaisanshu / santianliangjue / tiancantudou / weijie / xiaoqixiao / yuantong / zhouzi。

推荐结构：

```text
08大神蒸馏/
├── 大神索引.md
├── _安装记录.md
└── {作者拼音}.skill/
    ├── SKILL.md
    ├── references/
    └── scripts/
```

作者小 skill 必须自包含、可分发，不依赖本地 `02拆解报告` 路径，也不依赖固定安装位置。Claude Code 可安装到 `~/.claude/skills/`；其他 Agent 应安装到用户指定的 skill 根目录。

## 09文件归档

存放备份、待删除文件、旧版本 skill、迁移记录、治理记录。

清理默认移动到这里，不直接物理删除。
